Detailed Description

Header for Constant Wave Control. It provides APIs to use an Avian device in constant wave mode. All relevant parameters for the CW mode are set/get through these APIs. It also provides an interface to acquire data received through the RX antennas.

◆ ifx_device_cw_create()

IFX_DLL_PUBLIC ifx_Device_CW_Control_Handle_t ifx_device_cw_create ( ifx_Device_Handle_t  handle)

This function creates and initializes the constant wave mode controller instance.

The provided port within the device handle is used to detect the type of the connected device. According to the detected device type the CW mode controller is initialized. If the detected device is not recognized, an error is set.

Corresponding destroy method is not provided for CW mode. Reason is CW controller is part of device handle. In order to destroy a CW controller, application must destroy device handle.

After initialization the connection is still in deep sleep mode.

Parameters
[in]handleA device handle created before calling ifx_device_cw_create
Returns
A handle to the CW controller, used to control device for CW mode

)

This method sets the number of samples per Rx antenna to be captured in single shot.

All active Rx antennas have same number of samples. There is no need to pass antenna index.

The number of samples must be an even number. If samples is an odd number the error IFX_ERROR_ARGUMENT_INVALID is set.
