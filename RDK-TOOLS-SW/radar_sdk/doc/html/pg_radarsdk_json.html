Radar SDK Documentation
Infineon software Radar SDK for the development of applications with Infineon radar devices.
JSON Configuration

Configuration structure

The schema of the configuration (given in js-schema syntax) is 

{
    "?device": device_object,
    "device_config": device_config_object,
    "?presence_sensing": presence_sensing_object,
    "?segmentation": segmentation_object
}

where a starting question-mark (?) marks a property as optional. A valid configuration must contain the property device_config and may contain the properties device, presence_sensing, and segmentation. The schema of the objects device, device_config_object, presence_sensing_object, and segmentation_object are described in the next sections.

Description of properties:

  • device: Description for which radar devices the configuration is applicable.
  • device_config: Description of the device configuration.
  • presence_sensing: Configuration for the presence sensing application.
  • segmentation: Configuration for the segmentation application.

Schema of device

The schema of device is: 

{
    "types": Array.Of(String),
    "uuids": Array.Of(String)
}

Description of properties:

  • types [optional]: Array with types for which the configuration is applicable. Valid entries are "BGT60TR13" and "BGT60ATR24".
  • uuids [optional]: Array with unique ids (uuids) of applicable radar sensors.

The same uuid must not be an entry of types in different device sections. If the section device is missing, the corresponding configuration is treated as a default configuration if no other configuration is found for the given radar device.

Schema of device_config

The schema of device_config is: 

{
    "fmcw_scene": fmcw_scene_object,
    "fmcw_single_shape": chirp_timings_object,
    "fmcw_shape_set": shape_config_object
}

The schemas of fmcw_scene, fmcw_single_shape, and fmcw_shape_set are described in the following sections. The device_config object must contain exactly one of the objects fmcw_scene, fmcw_single_shape, and fmcw_shape_set. Setting two or more objects is not allowed.

Description of properties:

  • fmcw_scene: Configuration in terms of easy to understand high-level parameters.
  • fmcw_single_shape: Configuration of one single shape.
  • fmcw_shape_set: Configuration with full control over the shape configuration.

Schema of fmcw_scene

The fmcw_scene schema is: 

{
    "rx_antennas": Array.Of(Numbers),
    "tx_antennas": Array.Of(Numbers),
    "tx_power_level": Number,
    "if_gain_dB": Number,
    "range_resolution_m": Number,
    "max_range_m": Number,
    "max_speed_m_s": Number,
    "speed_resolution_m_s": Number,
    "frame_repetition_time_s": Number,
    "?sample_rate_Hz": Number,
    "?center_frequency_Hz": Number
}

Description of properties:

  • rx_antennas [required]: Array of integers with active RX antennas. E.g., to activate RX antennas 1 and 3: [1,3]
  • tx_antennas: Array of integers with active TX antennas. E.g., to activate TX antennas 1 and 3: [1,3]
  • tx_power_level [required]: The value determines the power of the emitted signal. Higher values correspond to higher power settings. The values do not directly correspond to a physical quantity. Allowed values are [0-31].
  • if_gain_dB [required]: Amplification factor in decibel that is applied to the IF signal coming from the RF mixer before it is fed into the analog-to-digital converter (ADC). Allowed values are [18-31].
  • range_resolution_m [required]: Defines the resolution of the radar device. That means the ability to resolve two objects with similar distance to the radar. The value is given in units of meters. The frequency difference of two adjacent bins in the range transform corresponds to range_resolution_m.
  • max_range_m [required]: Maximum distance to the radar that can be detected in units of meters. The bins of the range transform correspond to range values between 0 and max_range_m.
  • max_speed_m_s [required]: Maximum measurable radial velocity of a target from the radar sensor in units meters per second.
  • speed_resolution_m_s [required]: Desired speed resolution that one wants to achieve with micro-Doppler measurement. That means what velocity difference one can detect for two targets at the same distance from the radar. The value is in units of meters per second. the frequency difference of two adjacent bins of the Doppler transform correspond to speed_resolution_m_s.
  • frame_repetition_time_s [required]: Time between two consecutive frames in seconds and the inverse of the frame rate. Also known as frame period or frame time.
  • sample_rate_Hz [optional, default: 1MHz]: Sample rate of the analog-to-digital converter (ADC) in Hz.
  • center_frequency_Hz [optional]: Center frequency of the chirp in Hz. The center frequency is defined as the mid of the lower frequency (start frequency) and the upper frequency (end frequency) of the chirp. If omitted the default value for the center frequency from the radar device is used.

The upper and lower frequency of the FMCW ramp are computed from range_resolution_m and center_frequency_Hz. The number of ADC samples per chirp is computed from max_range_m. The number of chirps per frame is computed from speed_resolution_m_s. The pulse repetition time (PRT) is computed from max_speed_m_s.

Schema of fmcw_single_shape

The fmcw_single_shape schema is: 

{
    "rx_antennas": Array.Of(Numbers),
    "tx_antennas": Array.Of(Numbers),
    "tx_power_level": Number,
    "if_gain_dB": Number,
    "lower_frequency_Hz": Number,
    "upper_frequency_Hz": Number,
    "num_chirps_per_frame": Number,
    "num_samples_per_chirp": Number,
    "chirp_repetition_time_s": Number,
    "frame_repetition_time_s": Number,
    "?mimo_mode": String,
    "?sample_rate_Hz": Number
}

Description of properties:

  • rx_antennas [required]: Array of integers with activated RX antennas. E.g., to activate RX antennas 1 and 3: [1,3]
  • tx_antennas [required]: Array of integers with activated TX antennas. E.g., to activate TX antennas 1 and 3: [1,3]
  • tx_power_level [required]: The value determines the power of the emitted signal. Higher values correspond to higher power settings. The values do not directly correspond to a physical quantity. Allowed values are [0-31].
  • if_gain_dB [required]: Amplification factor in decibel that is applied to the IF signal coming from the RF mixer before it is fed into the analog-to-digital converter (ADC). Allowed values are [18-31].
  • lower_frequency_Hz [required]: Lower frequency of the FMCW chirp in Hz.
  • upper_frequency_Hz [required]: Upper frequency of the FMCW chirp in Hz.
  • num_chirps_per_frame [required]: Number of chirps in a single frame.
  • num_samples_per_chirp [required]: Number of samples in a single chirp (data frame).
  • chirp_repetition_time_s [required]: Time between the start of two consecutive chirps in a frame in seconds. Also known as chirp-to-chirp time or pulse repetition time.
  • frame_repetition_time_s [required]: Time between two consecutive frame in seconds and the inverse of the frame rate. Also known as frame period or frame time.
  • mimo_mode [optional, default: "off"]: Configuration of MIMO mode. Possible values are:
    • "off": MIMO is disabled.
    • "tdm": Time-division multiplexing. This value is currently only allowed with ATR24C with both transmitting antennas enabled.
  • sample_rate_Hz [optional, default: 1MHz]: Sample rate of the analog-to-digital converter (ADC) in Hz.

Schema of fmcw_shape_set

The fmcw_shape_set schema is: 

{
    "num_repetitions": Number,
    "following_power_mode": String,
    "end_delay_s": Number,
    "?sample_rate_Hz": Number,
    "shapes": Array.Of(shape_object)
}

Description:

  • num_repetition [required]: Number of repetitions of the shape set.
  • following_power_mode [required]: Power mode after end of the shape set. Valid values are "active", "idle", "deep sleep", and "deep sleep continue".
  • end_delay_s [required]: Delay after end of shape set in seconds.
  • shapes [required]: Array of shapes. The array must have at least one element and must not have more than 4 elements.
  • sample_rate_Hz [optional, default: 1MHz]: Sample rate of the analog-to-digital converter (ADC) in Hz.

The schema of shape_object is: 

{
    "type": String,
    "num_repetitions": Number,
    "following_power_mode": String,
    "end_delay_s": Number,
    "lower_frequency_Hz": Number,
    "upper_frequency_Hz": Number,
    "tx_power_level": Number,
    "hp_gain_level": Number,
    "vga_gain_level": Number,
    "up_chirp": chirp_object,
    "down_chirp": chirp_object
}

Description:

  • type [required]: Type of shape. Valid values are "up", "up-down", "down-up", "down".
  • num_repetitions [required]: Number of repetitions of the shape. This number must be a power of 2.
  • following_power_mode [required]: Power mode after end of the shape. Valid values are "active", "idle", "deep sleep", and "deep sleep continue".
  • end_delay_s [required]: Delay after end of shape in seconds.
  • lower_frequency_Hz [required]: Lower frequency of the FMCW chirp in Hz.
  • upper_frequency_Hz [required]: Upper frequency of the FMCW chirp in Hz.
  • hp_gain_dB [required]: Gain of high pass filter in dB. Allowed values are [18, 30]. The total gain is given as the sum of hp_gain_dB and vga_gain_dB.
  • vga_gain_dB [required]: Gain of integrated gain adjustment circuitry in dB. Allowed values are [0,5,10,15,20,25,30]. The total gain is given as the sum of hp_gain_dB and vga_gain_dB.
  • up_chirp [required]: Settings for up chirp. Setting is ignored if type is "down". In this case the setting can be omitted.
  • down_chirp [required]: Settings for down chirp. Setting is ignored if type is "up". In this case the setting can be omitted.

The schema of chirp_object is: 

{
    "rx_antennas": Array.Of(Number),
    "tx_antennas": Array.Of(Number),
    "num_samples": Number,
    "end_delay_s": Number
}

Description:

  • rx_antennas [required]: Array of integers with activated RX antennas. E.g., to activate RX antennas 1 and 3: [1,3]
  • tx_antennas [required]: Array of integers with activated TX antennas. E.g., to activate TX antennas 1 and 3: [1,3]
  • num_samples [required]: Number of samples in the chirp.
  • end_delay_s [required]: Delay after end of chirp in seconds.

Segmentation configuration

The schema for segmentation is given as: 

{
    "max_detection_range_m": Number,
    "segments_degrees": Array.Of(Number),
    "detection_sensitivity": String,
    "static_target_removal": String,
    "enable_hysteresis": Boolean,
    "max_num_tracks": Number
}

Description of properties:

  • max_detection_range_m [required]: Targets above this distance are ignored by the segmentation algorithm. The value is in units of meters.
  • segments_degrees [required]: Array of numbers that correspond to the widths of each segment in degree (from left to right). The field of view is the sum of all widths, the number of elements in the array is the number of segments.
  • detection_sensitivity [required]: Option to control the detection sensitivity. Allowed values are "low", "medium", or "high". "low" corresponds to sensor being less sensitive and hence less targets being detected. "high" corresponds to higher sensitivity of the sensor towards targets and hence more targets being detected.   • static_target_removal [required]: Allowed values are "low", "medium", or "high".
  • enable_hysteresis [required]: Flag indicating if hysteresis is enabled.
  • max_num_tracks [required]: Maximum number of tracks.

Presence Sensing

The schema for presence_sensing is: 

{
    "min_detection_range_m": Number,
    "max_detection_range_m": Number,
    "range_hysteresis_percentage": Number,
    "absence_confirm_count": Number,
    "presence_confirm_count": Number
}

Description:

  • min_detection_range_m [required]: Targets below this distance are ignored by the presence sensing algorithm. The value is in units of meters.
  • max_detection_range_m [required]: Targets above this distance are ignored by the presence sensing algorithm. The value is in units of meters.
  • range_hysteresis_percentage [required]: Hysteresis for max_detection_range_m while changing state from presence to absence and vice versa. The value are [0-100].
  • absence_confirm_count [required]: Number of consecutive absence detections in consecutive frames for making a definitive presence detection.
  • presence_confirm_count [required]: Number of consecutive presence detections in consecutive frames for making a definitive presence detection.

Example configurations

Using fmcw_scene

Example how to configure a radar chip using the scene configuration: Antennas RX1, RX2, RX3, and TX1 are activated, the range resolution is set to 15cm, the maximum range to 10m, the speed resolution to 0.2m/s, and the maximum speed to 5m/s. The frame repetition time (or frame period) is 0.1s and corresponds to a frame rate of 1/0.1s = 10Hz. The ADC sample rate has the default value of 1MHz. 

{
    "device_config": {
        "fmcw_scene": {
            "rx_antennas": [1, 2, 3],
            "tx_antennas": [1],
            "tx_power_level": 30,
            "if_gain_dB": 30,
            "range_resolution_m": 0.15,
            "max_range_m": 10,
            "max_speed_m_s": 5,
            "speed_resolution_m_s": 0.2,
            "frame_repetition_time_s": 0.1
        }
    }
}

Using fmcw_single_shape

Example how to configure a radar chip using the single shape configuration: MIMO mode is deactivated, the antennas RX1 and TX1 are activated, the chirp frequencies are set to 59GHz-61GHz. During each chirp 32 samples are taken and a frame consists of 128 chirps. The pulse repetition time is 0.001s (1ms) and the frame repetition time (or frame period) is 0.1s which corresponds to a frame rate of 10Hz. The ADC sample rate has the default value of 1MHz. 

{
    "device_config": {
        "fmcw_single_shape": {
            "rx_antennas": [1],
            "tx_antennas": [1],
            "tx_power_level": 30,
            "if_gain_dB": 30,
            "lower_frequency_Hz": 59e9,
            "upper_frequency_Hz": 61e9,
            "num_chirps_per_frame": 128,
            "num_samples_per_chirp": 32,
            "chirp_repetition_time_s": 0.001,
            "frame_repetition_time_s": 0.1
        }
    }
}

The following example is similar to the last one but with MIMO enabled. The MIMO mode is set to "tdm" (time-domain-multiplexing) and the TX antennas TX1 and TX2 are activated. This example only works with ATR24C. 

{
    "device_config": {
        "fmcw_single_shape": {
            "mimo_mode": "tdm",
            "rx_antennas": [1],
            "tx_antennas": [1,2],
            "tx_power_level": 30,
            "if_gain_dB": 30,
            "lower_frequency_Hz": 59e9,
            "upper_frequency_Hz": 61e9,
            "num_chirps_per_frame": 128,
            "num_samples_per_chirp": 32,
            "chirp_repetition_time_s": 0.001,
            "frame_repetition_time_s": 0.1
        }
    }
}

Using fmcw_shape_set

This example shows how to configure a radar chip using the shape set configuration. The shape set is repeated 32 times and after the shape set the chip goes into "deep sleep" power mode. Each shape set consists of one shape with an up-chirp and a down-chirp ("up-down"). In both up and down chirp consist 32 samples are taken and TX1, TX2, and RX1, RX2, RX3, and RX4 antennas are activated. After the shape the chip goes into "active" power mode. 

{
    "device_config": {
        "fmcw_shape_set": {
            "num_repetitions": 32,
            "following_power_mode": "deep sleep",
            "end_delay_s": 1e-6,
            "sample_rate_Hz": 1e6,
            "shapes": [
                {
                    "num_repetitions": 1,
                    "following_power_mode": "active",
                    "end_delay_s": 500e-9,
                    "type": "up-down",
                    "lower_frequency_Hz": 58e9,
                    "upper_frequency_Hz": 63e9,
                    "tx_power_level": 30,
                    "hp_gain_level": 30,
                    "vga_gain_level": 30,
                    "up_chirp": {
                        "rx_antennas": [1,2,3,4],
                        "tx_antennas": [1,2],
                        "num_samples": 32,
                        "end_delay_s": 10e-9
                    },
                    "down_chirp": {
                        "rx_antennas": [1,2,3,4],
                        "tx_antennas": [1,2],
                        "num_samples": 32,
                        "end_delay_s": 10e-9
                    }
                }
            ]
        }
    }
}

Segmentation configuration

This example shows how to combine the device config with the configuration for segmentation. 

{
    "device_config": {
        "fmcw_single_shape": {
            "sample_rate_Hz": 1e6,
            "mimo_mode": "off",
            "rx_antennas": [1],
            "tx_antennas": [1],
            "tx_power_level": 30,
            "if_gain_dB": 30,
            "lower_frequency_Hz": 59e9,
            "upper_frequency_Hz": 61e9,
            "num_chirps_per_frame": 128,
            "num_samples_per_chirp": 32,
            "chirp_repetition_time_s": 0.001,
            "frame_repetition_time_s": 0.1
        }
    },
    "segmentation": {
        "max_range_m": 8,
        "segments_degrees": [30, 30, 30],
        "detection_sensitivity": "medium",
        "static_target_removal": "medium",
        "enable_hysteresis": true,
        "max_num_tracks": 6
    }
}

This example shows how to provide different configurations for different radar devices. The configurations are given as an array and the section device specifies for which types or uuids the configuration is valid. If the section device is missing, this configuration is treated as the default configuration in case no other matching configuration is found.

    {

            "uuids": ["01234567-89ab-cdef-0123-456789abcdef"]

                "tx_antennas": [1],

                "max_speed_m_s": 5,
                "speed_resolution_m_s": 0.2,
                "frame_repetition_time_s": 0.1
            }
        }
    },
    {
        "device": {
            "types": [ "BGT60TR13" ],
            "uuids": [ "00034567-89ab-cdef-0123-456789abcdef", "01134567-89ab-cdef-0123-456789abcdef" ]

        },
        "device_config": {
            "fmcw_scene": {
                "rx_antennas": [1, 2, 3],
                "tx_antennas": [1],
