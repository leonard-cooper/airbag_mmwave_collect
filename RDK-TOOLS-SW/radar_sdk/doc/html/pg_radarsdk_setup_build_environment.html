Radar SDK Documentation
Infineon software Radar SDK for the development of applications with Infineon radar devices.
Setup Build environment guidelines

The following compilers and platforms are supported:

  • Windows, amd64 (Visual Studio 2017 or higher)
  • Windows, MinGW, amd64 (GCC)
  • Linux, amd64 (Ubuntu 20.04, GCC)
  • Linux, armhf (Raspbian Buster on Raspberry Pi, GCC)
  • Linux, aarch64

Radar SDK on Windows

To build the Radar SDK applications on Windows, one of the following approaches can be applied as below:

MinGW

MinGW is a free and open source software development environment. It includes a portation of the GNU compiler collection to Windows and allows to build libraries and programs for Microsoft Windows.

Setup build environment

You need to install the MinGW environment, cmake, and ninja:

  • We suggest to download the MinGW distribution available at winlibs.com. Download the latest distribution for 64bit Windows (Architecture: x86_64) and extract the archive. In the following we assume that the distribution has been extracted to C:\mingw64\.
  • Download the latest version of cmake and install it. In the following we assume that cmake has been installed to C:\Program Files\CMake\.
  • Download the latest version of ninja and copy the execeutable ninja.exe into the directory C:\mingw64\bin\.
Build Radar SDK Applications

To build the project under Windows, please follow the following instructions:

  • Open a Windows command prompt (cmd) and change into the directory of the RDK release package.
  • Add the directory c:\mingw64\bin to the PATH: 
    $ SET PATH=c:\mingw64\bin;%PATH%
  • Create a new folder build and set it as working directory: 
    $ mkdir build/
$ cd build
  • Generate build files for Ninja using the cmake command: 
    $ "c:\Program Files\CMake\bin\cmake.exe" -G Ninja ..
  • Build executable programs and libraries from RDK source code using ninja command: 
    $ ninja

The compiled executables and libraries have been created in the directory build\bin. For using the executables *.exe and dynamic link libraries *.dll generated by using the MinGW environment, the PATH adjustment, as mentioned above, must be effective in the usage scope.

Visual Studio

Download and install Visual Studio

The latest version of Visual Studio (32bit and 64bit) can be downloaded from Downloads page from Microsoft. Please follow the steps on the Install Visual Studio page to to install Visual Studio on your PC.

Prepare build environment

To generate the RDK Visual Studio Solution File (*.snl), you can use the CMake GUI. Please download the latest version of CMake from the Download page from cmake.org.

  • Run the CMake (cmake-gui) application.
  • Give the right directory paths:
    • Where is the source code: e.g. "C:/radar_sdk"
    • Where to build the binaries: e.g. "C:/radar_sdk/vs-build"
  • Click Configure and Generate in an order.

The Visual Studio solution file (*.snl) will be generated in the directory "vs-build" .

Build Radar SDK Applications

Open the solution file (*.snl) from Visual Studio, you can start from there.

WSL (Windows Subsystem for Linux)

Download and install WSL

Before you can install any Linux distribution on it, WSL has to be enabled in one of the following ways:

Using the GUI for enabling Windows features

  • Open the Start Menu and search "Turn Windows features on or off"
  • Select Windows Subsystem for Linux
  • Click OK
  • Restart your computer when prompted

Using PowerShell

  • Open PowerShell as Administrator and run: 
    Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux
  • Restart your computer when prompted

For more details please refer to the following links: Installation Instructions for WSL. Recommend to use Ubuntu 20.04 distribution.

Radar SDK on Linux

In the following, it is assumed that the operating system is Ubuntu 20.04 (amd64), or Raspberry Pi (Raspbian GNU/Linux 10 Buster) is used.

To build the Radar SDK applications on Linux:

Prepare build environment

Before you start, make sure that all packages are up to date by executing the following command on bash shell: 

$ sudo apt update
$ sudo apt upgrade
$ sudo apt install cmake make gcc build-essential
Build Radar SDK Applications
  • Set working directory to folder where the RDK release package is extracted (e.g. "/home/username/radar_sdk"): 
    $ cd ~/radar_sdk
  • Create a new folder build, and step inside to it as working directory: 
    $ mkdir build/
$ cd build
  • Generate Makefiles and build the example apps: 
    $ cmake ..
$ make

The built executables are located inside in the directory build/ .

Error opening the radar device

If you cannot open the radar device or you get an IFX_ERROR_NO_DEVICE error, make sure that you can access the device file /dev/ttyACM0 (the number might be different on your computer). On Ubuntu and Raspbian, make sure that your current user is in the group dialout. You can check the groups of your current user using: 

$ groups

To add your user to the group dialout, run 

$ sudo usermod -a -G dialout username

and replace username by your actual user name. Please note that the group assignment only takes effect after logging out of all running sessions.

Linker errors on Raspberry Pi

If you encounter errors like: 

libradar_sdk.so: undefined reference to `logf@GLIBC_2.27'

while linking you most likely are not using Raspbian 10 but Raspbian 9. In this case, please upgrade to Raspbian 10 (Buster).

You can check which Raspian version you are running using the command lsb_release: 

$ lsb_release -a
No LSB modules are available.
Distributor ID: Raspbian
Description:    Raspbian GNU/Linux 10 (buster)
Release:        10
Codename:       buster

Policy and Guidelines

ABI support for Linux

Ubuntu 20.04 LTS is fully compliant as is Debian Buster.

We support glibc versions 2.28 to at least 2.31. We support libstdc++ versions 6.0.28 to at least 6.0.35.
